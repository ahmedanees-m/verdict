# VERDICT Acceptance rules (committed before building the checks)

- Every eval case declares an expected verdict AND the check expected to fire.
- No number reaches output without a receipt; missing data => INSUFFICIENT (never a fabricated value).
- The direction check fires ONLY when the claim is context-matched (CD4+ T + matching condition) AND well-powered.
- SUPPORTED verdicts are COMPUTED from receipted evidence, never retrieved (deleted-knowledge test).
- No case uses DEXI/CLEC16A as a verdict target (unresolved controversy).
- All verdicts are scoped to "given CD4+ T-cell data", never "the target is wrong".
- The fabrication-impossible test stays green.
- The confusion matrix over EVAL_SET is reported in the submission.
